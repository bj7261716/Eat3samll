## ⚡ 快速測試清單

### 1. 基礎功能測試（無需 API 金鑰）

- [ ] 頁面載入正常，顯示搜尋表單
- [ ] 料理類型顯示 10 個選項（含 emoji）
- [ ] 價格選項顯示 4 個等級（$~$$$$）
- [ ] 距離滑桿可拖動，數值即時更新

### 2. 響應式測試

使用 Chrome DevTools (F12) 測試：

- [ ] 手機版 (375px)：單欄布局
- [ ] 平板版 (768px)：雙欄布局  
- [ ] 桌面版 (1920px)：三欄布局

### 3. API 功能測試（需要設定 API 金鑰）

**設定步驟：**
1. 開啟 `js/config.js`
2. 第 15 行：將 `YOUR_GOOGLE_MAPS_API_KEY_HERE` 替換為您的金鑰
3. 重新整理頁面

**測試項目：**
- [ ] 設定搜尋條件後點擊「搜尋美食」
- [ ] 顯示 3-5 家餐廳卡片
- [ ] 切換到地圖視圖，標記正確顯示
- [ ] 點擊餐廳卡片的「導航」按鈕，開啟地圖應用
- [ ] 重新整理頁面，上次條件自動載入

## 🐛 常見問題

### 頁面顯示 API 金鑰警告
→ 正常現象，設定金鑰後即可使用完整功能

### 地圖無法載入
→ 檢查 API 金鑰是否正確，並確認已啟用 Maps JavaScript API

### 找不到餐廳
→ 允許瀏覽器地理位置權限，或嘗試增加距離範圍

### 樣式跑版
→ 確認所有 CSS 檔案都已正確載入
